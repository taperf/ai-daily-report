## 📝 OFFICIAL BLOGS

**Anthropic Engineering: An Update on Recent Claude Code Quality Reports**

Anthropic published a detailed postmortem on Claude Code quality degradation reports over the past month. They traced the issues to three separate changes: (1) On March 4, they lowered default reasoning effort from high to medium to reduce latency — wrong tradeoff, reverted April 7; (2) On March 26, a bug in session thinking cleanup caused Claude to seem forgetful and repetitive — fixed April 10; (3) On April 16, a verbosity-reduction prompt change hurt coding quality — reverted April 20. Because each change hit different traffic slices on different schedules, the aggregate effect looked like broad, inconsistent degradation. Importantly, the API and inference layer were never affected. Going forward, Anthropic promises better monitoring to catch these issues earlier.

Anthropic 发布了关于 Claude Code 质量下降报告的详细事后分析。问题追溯到三个独立变更：(1) 3月4日将默认推理强度从 high 降为 medium 以减少延迟——错误取舍，4月7日回滚；(2) 3月26日会话思考清理的 bug 导致 Claude 显得健忘和重复——4月10日修复；(3) 4月16日减少冗余的 prompt 变更伤害了编码质量——4月20日回滚。由于三个变更影响了不同流量切片和不同时间段，综合效果看起来像广泛而不一致的退化。重要的是，API 和推理层未受影响。

**Claude Blog: Built-in Memory for Claude Managed Agents**

Memory on Claude Managed Agents is now in public beta. Agents can learn from every session using filesystem-based memory — memories are stored as files, so developers can export, manage via API, and maintain full control. Key features: scoped permissions, audit logs, concurrent multi-agent access without overwrites, rollback capability, and version tracking. Memory stores can be shared across agents with different access scopes (e.g. org-wide read-only + per-user read/write).

Claude Managed Agents 的内置记忆功能进入公测。Agent 可以通过基于文件系统的记忆层从每次会话中学习——记忆以文件形式存储，开发者可以导出、通过 API 管理并保持完全控制。核心特性：分级权限、审计日志、多 agent 并发访问无覆盖、回滚能力和版本追踪。

🔗 https://claude.com/blog/claude-managed-agents-memory
