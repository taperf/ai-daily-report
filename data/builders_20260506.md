## 📝 OFFICIAL BLOG

### Anthropic Engineering: Claude Code auto mode — a safer way to skip permissions

**The Takeaway:** Anthropic推出Claude Code的"auto mode"——一种基于模型分类器的权限委托机制，在完全手动审批和无审批之间找到中间地带。

**核心内容：**
- 默认情况下Claude Code每次执行命令或修改文件都需用户批准，但93%的请求都被通过→审批疲劳
- 此前只有两个极端选项：沙箱隔离（安全但维护成本高）或`--dangerously-skip-permissions`（零维护但不安全）
- **Auto mode**用模型分类器自动判断操作是否安全，目标是拦截不符合用户意图的危险操作，放行其余操作
- 内部事故记录包括：删除远程git分支、上传工程师GitHub auth token到内部集群、对生产数据库执行迁移——都是模型"过度热心"的结果
- 相关内容已记录在Claude Opus 4.6 system card §6.2.1和§6.2.3.3中

**Why it matters:** 这解决了AI编码助手最大的UX摩擦之一。随着Agent越来越自主，"何时该问用户、何时该自己决定"成为核心设计问题。Anthropic选择用AI来判断AI的行为边界，是一个优雅但需要持续迭代的方案。

> 🔗 https://www.anthropic.com/engineering/claude-code-auto-mode
