### Boris Cherny (Claude Code, Anthropic)

Cherny laid out a four-level framework for engineering teams adopting Claude Code. Level 1 is basic usage; Level 2 is tracking adoption; Level 3 is giving Claude ways to verify its own work end to end (auto mode for permissions, automated code review, security review, multi-agent interfaces like Agent view in CLI/Desktop/iOS/Android). Level 4 — where he says he personally is — means using features like /loop, /batch, dynamic workflows, and worktree isolation for subagents. He stressed that the "bigger payoff comes when fixing and maintaining happens in the background and your teams can focus on building." For tracking ROI, he suggested measuring not just usage (activity) but asking: would you have spent engineering effort on this anyway? If yes, how much and what would it have cost in manual eng-hours?

Cherny 梳理了工程团队采用 Claude Code 的四级框架。第一级是基础使用；第二级是追踪采用率；第三级是让 Claude 能端到端验证自己的工作（自动权限模式、自动代码审查、安全审查、多 Agent 界面如 CLI/Desktop/iOS/Android 的 Agent view）。第四级——他说自己目前就在这一级——意味着使用 /loop、/batch、动态工作流和子 Agent 的 worktree 隔离。他强调"真正的回报在于修复和维护在后台自动完成，团队可以专注于构建"。关于 ROI 追踪，他建议不要只看使用量（活动指标），而要问：这件事本来就要花工程人力吗？如果是，要花多少、相当于多少手动工程师工时？

### Madhu Guru (Sr Director, AI at Meta; ex-Google Gemini, Veo, Nano)

Guru outlined three imperatives for enterprises in the open-weight model era: 1) Build rigorous, fast-to-run evals (regression + aspirational) — "eval velocity is a competitive advantage"; 2) Build your own model router, because "nobody understands your business and users like you do"; 3) Build a model-agnostic harness that normalizes prompts, context management, tool definitions, and output parsing across models, making it easy to switch once your evals pass.

Guru 为企业列出了开源权重模型时代的三条铁律：1）构建严格、快速运行的评估体系（回归测试 +  aspirational 测试）——"评估速度本身就是竞争优势"；2）自建模型路由，因为"没有人比你更懂自己的业务和用户"；3）构建模型无关的 harness，统一 prompts、上下文管理、工具定义和输出解析，这样一旦评估通过就能无缝切换模型。
