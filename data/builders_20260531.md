## 🎙️ PODCASTS

**No Priors — "Building an AI Guardian for Enterprise" with Onyx Security CEO Maxim Bar Kogan**

🎯 The Takeaway: As AI agents become exponentially more autonomous in enterprises, traditional security tools (identity, endpoint, API) are fundamentally inadequate — a new category of "AI guardians" using specialized small models to oversee other agents is emerging as a critical infrastructure layer.

Maxim Bar Kogan, co-founder and CEO of Israel-based Onyx Security, describes building "agents to watch the AI agents." The timing thesis: when AutoGPT appeared in 2023, it was clear that truly autonomous agents would eventually need oversight, but the market wasn't ready. Now, with Claude Code, OpenClaw, and similar tools seeing explosive enterprise adoption (Anthropic's revenue largely comes from enterprises paying for Claude Code), the security gap is urgent.

The core challenge: existing security approaches fail because AI agents need broad permissions to be useful (you want Claude Code to have your full access so it can work while you're at lunch), and traditional proxy/policy engines can't determine *intent* — whether an agent deleting a database is executing your instruction or hallucinating. Onyx's solution: train small, specialized models that are "good at only one thing" — detecting when a smarter agent should review an action. This mirrors blitz chess intuition: fast pattern-matching for 99% of moves, deep calculation only at critical junctures.

Key market insight: in typical enterprise deployments, ~50% of AI agent usage is autonomous coding agents (Claude Code leading), ~45% is low-code automations (growing fastest with tools like Claude Co-worker and OpenClaw gaining enterprise sanction), and only ~2% is custom-built first-party agents. The autonomous category is growing fastest but "typically comes without any controls."

Maxim Bar Kogan（以色列初创公司 Onyx Security 的联合创始人兼 CEO）在 No Priors 播客中深入讨论了「用 AI 监管 AI」的命题。随着 Claude Code、OpenClaw 等自主 agent 在企业中爆发式增长，传统安全工具（身份验证、终端安全、API 安全）根本无法应对——因为 AI agent 需要广泛权限才能发挥作用，而传统工具无法判断 agent 的「意图」（删数据库是在执行你的指令，还是在幻觉？）。

Onyx 的方案：训练极小的专用模型，只做一件事——判断「现在是否需要更智能的 agent 来审查这个动作」。这类似闪电棋的直觉判断：99% 的动作靠快速模式匹配，只在关键节点启动深度计算。

市场数据：典型企业中约 50% 的 AI agent 使用量来自自主编码 agent（Claude Code 领先），45% 是低代码自动化（Claude Co-worker 和 OpenClaw 增长最快），仅 2% 是自建 agent。自主类别增长最快，但「通常没有任何控制措施」。
