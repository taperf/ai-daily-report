## 🎙️ Podcast

### The MAD Podcast: "Why Every AI Agent Needs Its Own Computer" — Ivan Burazin (Daytona)

Matt Turck 对话 Daytona CEO Ivan Burazin，深入探讨为什么每个AI智能体都需要自己的计算机（sandbox）。

**Key insights / 核心观点：**

1. **Agents as digital knowledge workers / 智能体即数字知识工作者**：Just as humans need computers to be productive, agents need their own sandboxed environments to install tools, access resources, and operate autonomously.

2. **Why Daytona ditched Kubernetes / 为何抛弃K8s**：Standard container orchestration wasn't designed for the bursty, ephemeral workload patterns of AI agents. They built a custom scheduler optimized for rapid spin-up and tear-down.

3. **The full agent stack / 完整智能体技术栈**：Ivan breaks it down — models → sandboxes → tools → MCP → memory → orchestration. Each layer is evolving independently and rapidly.

4. **Coming GPU/CPU shortage / 即将到来的算力短缺**：The current "super cycle" won't last forever; companies that pause are effectively timing the market. Infrastructure demand is growing faster than supply.

5. **Security first / 安全优先**：The original motivation for sandboxes was isolation — you'd never give an agent direct access to your bank account. The sandbox boundary is the trust boundary.
