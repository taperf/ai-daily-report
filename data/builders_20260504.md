## 📝 OFFICIAL BLOG

### Anthropic Engineering: Scaling Managed Agents — Decoupling the Brain from the Hands

Anthropic工程博客发布重磅技术文章，详细阐述了Managed Agents的架构设计哲学。核心思想：将Agent的「大脑」（Claude + harness控制循环）与「双手」（沙箱和工具执行环境）以及「会话日志」彻底解耦。

**关键洞察：**
- Harness编码了「Claude不能独立完成什么」的假设，但这些假设会随模型进化而过时（例如Sonnet 4.5的「上下文焦虑」在Opus 4.5上已消失）
- 借鉴操作系统虚拟化思路：就像read()命令不关心底层是70年代磁盘还是现代SSD，Agent的session/harness/sandbox三层接口设计为实现可自由替换
- 早期「宠物」架构（所有组件在单容器内）导致调试困难和客户VPC集成受阻
- 解耦后：harness可独立部署、sandbox可连接客户环境、session作为append-only日志独立存在

这篇文章本质上是Anthropic对「AI Agent基础设施应该怎么建」的架构宣言——与今日AI行业日报中「Harness剥离沙箱」的技术讨论完美呼应。

Anthropic Engineering published a deep architecture post on Managed Agents. Core idea: decouple the "brain" (Claude + harness) from the "hands" (sandboxes/tools) and "session" (event log). Key insight: harness assumptions go stale as models improve. They virtualized agent components following OS design principles—interfaces that outlast implementations.

🔗 https://www.anthropic.com/engineering/managed-agents
