## 🎙️ Podcasts

**Training Data: Anthropic's Katelyn Lesse & Angela Jiang — Building an Ecosystem, not a Walled Garden**

Anthropic 平台负责人 Katelyn Lesse 和 Angela Jiang 在 Training Data 播客中深入探讨了 Claude 平台的演进路线。核心观点：Anthropic 的 Platform 团队同时服务内部（产品基础设施）和外部（开发者 API），坚持同一个底层哲学——让任何 builder 都能用 Claude 构建任何想要的东西。

两人揭示了 Anthropic 的抽象层级路线图：从知识层（knowledge）到执行层（execution），再到更高层的协调层（coordination）。在协调层，他们正在构建"策略（strategies）"——一种 meta harness，让 token 拥有不同的"工作（jobs）"：有些 token 负责建议，有些负责执行，可以组合编排。

Katelyn 指出，当前 builder 做 agent 时几乎只有两个杠杆：换更大的模型，或者让模型跑更久。但还有第三个被低估的杠杆：best-of-N 策略——同一任务并行跑多个 agent，取最优结果。虽然这个概念在论文里常见，但将其产品化并投入实际使用极其困难。这正是 Anthropic 正在攻克的难题。

Angela Jiang（前 Stripe 工程师）谈到了成本管理的演化。她认为行业正在经历一个自然周期：从"随便用"到精细化管控 token 成本。她拿 Stripe 时期关注 AWS 账单的例子类比，说明企业需要建立 guardrails 来发现和优化不合理的 AI 支出。但她也强调，不应该简单地设置一个硬封顶（cap），而是应该鼓励创新，同时从侧面观察哪些结果可以通过更聪明的方式来达成。

最值得关注的是关于生态系统的立场。两人明确表示，Anthropic 不相信"一个 harness 通配所有模型"的 router 思路，而是认为 harness 和 agentic layer 应该与特定模型家族调优配合。他们希望成为 Claude 生态的推动者，而非围墙花园的建设者。这与 Vercel 最近推出的 Harness Agent（绑定模型家族的 harness）形成了有趣的呼应。
