## 🔥 Andrej Karpathy (@karpathy)
*Director of AI @ Tesla (prev), OpenAI founding team*

Karpathy shared highlights from his Sequoia Ascent 2026 fireside chat, pushing three provocative themes about LLMs going beyond "speeding up what existed":

1. **Apps fully engulfed by LLMs** — like "menugen" (image in → image out, zero classical code needed)
2. **Install .md instead of .sh** — Why write complex bash scripts when you can write English instructions for an LLM interpreter that adapts to your setup?
3. **LLM knowledge bases** — computation over unstructured knowledge from arbitrary sources, something *impossible* with classical code

He also wrestled with "jaggedness" — how the same model can coherently refactor 100K lines of code while giving absurd advice about car washes. His explanation: it's about verifiability + economics (training data distribution is dictated by revenue/TAM). You're either on the RL rails and flying, or off-roading with a machete.

Final theme: the **agent-native economy** — decomposing products into sensors, actuators, and logic across computing paradigms 1.0/2.0/3.0, with hints of fully neural computing where CPUs become coprocessors.

Karpathy 分享了他在 Sequoia Ascent 2026 炉边谈话的要点，提出三个 LLM 超越「加速现有事物」的新方向：1）完全由 LLM 吞噬的应用（无需传统代码）；2）用 .md 文件代替 .sh 脚本（让 LLM 当英语解释器）；3）LLM 知识库处理非结构化数据（传统代码根本做不到）。他还探讨了 LLM 的「参差不齐」现象——同一模型能重构10万行代码却给出荒谬建议，原因在于可验证性+经济学（训练数据分布由营收/TAM决定）。最后展望了 Agent 原生经济体。

## 🏢 Aaron Levie (@levie)
*CEO @Box*

Deep analysis on **software pricing in the agent era**: As agents become the biggest users of software, all software must go headless — agents won't use your UI, they'll talk to your APIs.

Three pricing models emerging:
1. **Seats for people remain** — but must include API usage allowance for agents acting on behalf of users. If you don't enable this, you're DOA.
2. **Agent "seats"** — for stateful agents with their own workspace/permissions, but pricing can't mirror human seats (one company might use 1 agent, another 1,000)
3. **Consumption model** — dominant for headless use above seat allotment. May need new "outcome-based" API calls rather than raw API sequences.

His conclusion: growth is effectively unbounded as agent use-cases dramatically exceed what people do with tools today. Every platform must go headless or die.

Levie 深度分析了 Agent 时代的软件定价逻辑：所有软件必须无头化（headless），Agent 不会用你的 UI，它们只和 API 对话。三种定价模式浮现：人类座位制（必须附带 Agent 代理调用额度）、Agent 座位制（有状态的 Agent 工作空间）、消费制（按量计费，将成为主流）。结论：不走 headless 路线的平台必死。

## 🎙️ Podcast: Demis Hassabis — Building DeepMind, AlphaFold, and the Final Stretch to AGI

*Sequoia Capital fireside chat*

DeepMind CEO Demis Hassabis 在 Sequoia 炉边谈话中回顾了从国际象棋神童到游戏创业（Theme Park）到神经科学再到 DeepMind 的完整路径。核心观点：

**AI for Science**: AlphaFold 已是一个里程碑时刻，下一步是药物设计（通过 Isomorphic Labs）。未来2-3年AI将在材料科学、环境、能源领域产生巨大影响。Demis 认为 AI 作为「虚拟科学家」做实验的成本将从数百万美元降至接近零。

**AGI 时间线**: 2030年，他一直保持这个预测。DeepMind 2010年创立时就设定为20年任务，目前基本精确在轨。

**信息即基础**: Hassabis 认为信息可能比能量和物质更基本——宇宙本质上是信息处理系统，这让 AI 的意义比表面看起来更深远。

**意识与工具**: 建议先把 AGI 作为工具建好，再跨越下一个「卢比孔河」（自主性/意识）。用工具级 AGI 来帮助回答关于意识的深层问题。

**选择合作伙伴**: 如果玩策略游戏要选一位历史科学家当队友？冯·诺依曼——因为需要一个博弈论专家。

Demis Hassabis discussed his path from chess prodigy → games → neuroscience → DeepMind. Key points: AGI by 2030 (on track with 2010 prediction); information may be more fundamental than energy/matter; build AGI as a tool first, then address consciousness; AlphaFold as proof classical machines can model quantum systems; AI will reduce scientific experiment costs to near-zero.
